# 🧬 Guestbook Node Logic Report

이 문서는 IYOHOUSE 방명록(Guestbook) 네트워크 모델의 노드 크기 계산 및 물리 엔진 로직을 정리한 기술 문서입니다.

## 1. 노드 크기 차등화 (Sizing Logic)
노드들은 등록된 시간의 최신순(Recency)에 따라 3단계로 크기가 결정됩니다.

| 순위 (Rank) | 비중 | 데스크탑 크기 | 모바일 크기 (60%) |
| :--- | :--- | :--- | :--- |
| **최신 노드** | 상위 10% | 120px | 72px |
| **일반 노드** | 상위 10% ~ 60% | 70px | 42px |
| **오래된 노드** | 하위 40% | 40px | 24px |

- **계산 방식:** 전체 노드 중 최신일수록 높은 순위를 매겨 백분율(`percentile`)을 산출합니다.
- **구현 위치:** `src/app/page.tsx` 내 `drawNetwork` 루프에서 매 프레임 계산됩니다.

## 2. 반응형 및 모바일 최적화 (Mobile Scaling)
모바일 환경에서의 가동률과 공간 효율을 극대화하기 위해 다음 로직이 적용되었습니다.
- **감지 로직:** `window.innerWidth < 768` 조건을 통해 모바일 환경을 판단합니다.
- **스케일링:** 모든 노드의 `targetSize`에 `0.6` 상수를 곱하여 부피를 줄입니다.
- **글자 크기:** CSS `clamp()`를 통해 모바일에서도 제목이 상자 밖으로 나가지 않도록 조절합니다.

## 3. 호버(Hover) 및 안정화 로직 (Jitter Prevention)
마우스를 올렸을 때 상자가 떨리는 현상(Jitter)을 방지하기 위한 핵심 물리 제어 로직입니다.

- **물리력 동결:** 특정 노드가 호버링(`isHovered`)되면, 해당 노드의 좌표 이동 로직(`x`, `y` 업데이트)을 즉시 중단하여 제자리에 고정합니다.
- **반발력 예외 처리:** 호버된 노드는 다른 노드가 밀어내는 힘(Repulsion)을 무시하므로 주변 환경에 의해 튕겨나가지 않습니다.
- **동적 높이 확장:** `container-type: size` 속성을 제거하여, 텍스트 내용이 길어지면 CSS `height: auto`에 의해 상자가 아래로 자연스럽게 늘어나도록 설계되었습니다.

## 4. 디자인 및 타이포그래피 (UI/UX)
- **폰트:** 가독성과 현대성을 위해 **Noto Sans KR**을 프로젝트 전체 및 방명록에 적용했습니다.
- **정렬:** 제목, 본문, 날짜 요소 모두를 수평 중앙 정렬(`text-align: center`)하여 안정감을 주었습니다.
- **디테일:** 호버 시에만 **검정색 테두리와 둥근 모서리**가 나타나도록 하여 인터랙션을 강조했습니다.
