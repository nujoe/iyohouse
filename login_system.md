# IYOHOUSE Login & Workshop Registration System Proposal

이 문서는 이요하우스 워크숍 신청 시스템의 레이아웃 및 기능 개선을 위한 로그인 시스템과 데이터 동기화 방안을 정리한 초안입니다.

## 1. 개요 (Background)
*   **문제점**: 선착순(FCFS) 워크숍 신청 시 사용자가 정보를 입력하는 시간이 길어지면 데이터가 누락되거나 신청에 실패할 확률이 높음.
*   **해결책**: SNS 로그인을 도입하고, 가입 직후 필수 정보를 미리 받아두어 신청 시에는 '결제'에만 집중하도록 함.

---

## 2. 설계 목표 (Design Goals)
*   **SNS 로그인 (Google/Kakao)**: 가입 허들을 최소화.
*   **필수 온보딩 (Onboarding)**: 이름, 전화번호, 자기소개를 가입 시 필수 입력.
*   **구글 시트 실시간 연동**: 관리자가 DB를 직접 보지 않고도 워크숍별로 신청자 현황을 실시간 파악.

---

## 3. 상세 설계 제안 (Design Approaches)

### A. 로그인 및 온보딩 플로우 (User Auth)
*   **옵션 1: 미들웨어 기반 강제 온보딩 (추천)**
    *   사용자가 로그인 후 `profiles` 테이블에 필수 정보가 없으면 자동으로 `/onboarding` 페이지로 이동.
    *   정보를 입력하기 전까지는 다른 페이지 이동을 제한하여 데이터 누락을 원천 차단.
*   **옵션 2: 신청 시점 프로필 체크**
    *   로그인은 자유롭게 하되, '워크숍 신청' 버튼 클릭 시 프로필 정보가 없으면 입력 폼을 띄움.
    *   사용자 흐름을 덜 방해하지만, FCFS 상황에서 정보 입력 시간만큼 불리해질 수 있음.

### B. 관리자 뷰 및 데이터 동기화 (Admin & Sync)
*   **구글 시트 실시간 연동**
    *   결제가 완료되는 시점(`api/payment/confirm`)에서 Google Sheets API를 호출.
    *   **워크숍별 탭 분리**: 워크숍 ID 또는 제목을 기반으로 해당 시트의 서브 탭에 데이터를 추가.
    *   **포함 데이터**: 신청 시간, 이름, 연락처, 이메일, 결제 금액, 자기소개 등.

---

## 4. 데이터베이스 구조 (Database Schema)

### `profiles` (Supabase Public Table)
| Column | Type | Description |
| :--- | :--- | :--- |
| id | uuid (PK) | auth.users.id 참조 |
| full_name | text | 실명 |
| phone | text | 연락처 |
| bio | text | 자기소개/분야 |
| updated_at | timestamp | 수정 일시 |
